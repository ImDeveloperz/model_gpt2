Pour installer une machine virtuelle avec SunOS (anciennement Solaris), configurer le réseau, et permettre la communication entre deux machines (un serveur et un client), vous pouvez suivre les étapes suivantes :
1. Préparation de l'Environnement

    Logiciel de virtualisation : Vous aurez besoin d'un logiciel de virtualisation tel que VirtualBox ou VMware.
    Image SunOS : Téléchargez l'image ISO de SunOS (Solaris) sur le site officiel d'Oracle ou à partir d'autres sources autorisées.
    Machines virtuelles : Créez deux machines virtuelles, une pour le serveur et une autre pour le client.

2. Installation de SunOS sur les Machines Virtuelles
Étape 1 : Création de la Machine Virtuelle

    Ouvrez le logiciel de virtualisation (VirtualBox ou VMware).
    Créez une nouvelle machine virtuelle :
        Nom : Choisissez un nom (par exemple, "SunOS_Server").
        Type : Unix.
        Version : Solaris (x64 ou x86 selon votre image ISO).
    Configurer les ressources :
        Mémoire RAM : Allouez au moins 2 Go de RAM.
        Disque dur : Créez un disque virtuel (20 Go ou plus).
    Attachez l'image ISO de SunOS téléchargée au lecteur de DVD virtuel de la machine.

Étape 2 : Installation de SunOS

    Démarrez la machine virtuelle et lancez l'installation depuis l'image ISO.

    Suivez les instructions à l'écran pour installer SunOS. Choisissez les paramètres par défaut si vous n'êtes pas sûr des options.

    Configurez le réseau pendant l'installation pour utiliser un mode "NAT" ou "Bridge Adapter" :
        NAT : Permet à la machine virtuelle d'accéder à Internet via l'hôte.
        Bridge Adapter : Associe l'interface réseau de la VM à celle de l'hôte, permettant à la machine virtuelle d'obtenir une adresse IP sur le même réseau que l'hôte.

    Complétez l'installation et redémarrez la machine virtuelle.

3. Configuration Réseau
Étape 1 : Attribution des Adresses IP

    Configurer l'adresse IP de la machine serveur :
        Connectez-vous à la machine serveur (SunOS_Server).
        Utilisez les commandes suivantes pour configurer l'adresse IP :

        bash

        ifconfig -a          # Afficher les interfaces réseau disponibles
        ifconfig [interface] plumb  # Activer l'interface réseau
        ifconfig [interface] 192.168.1.10 netmask 255.255.255.0 up  # Exemple d'adresse IP

    Configurer l'adresse IP de la machine client :
        Suivez les mêmes étapes que pour le serveur, mais avec une adresse IP différente, comme 192.168.1.11.

Étape 2 : Vérification de la Connectivité

    Pinger l'adresse IP du serveur à partir de la machine client :

    bash

    ping 192.168.1.10

    Si la réponse est reçue, cela signifie que les deux machines peuvent communiquer.

4. Installation des Services de Communication

Pour permettre la communication entre le serveur et le client via des services tels que SSH ou Telnet :
Étape 1 : Installation d'OpenSSH (Optionnel)

    Installer le service SSH sur SunOS :
        Si non installé par défaut, téléchargez le package OpenSSH.
        Installez-le avec pkgadd ou une commande équivalente.

    Démarrer le service SSH :

    bash

    svcadm enable ssh

Étape 2 : Test de la Connexion SSH

    Depuis la machine client, essayez de vous connecter au serveur via SSH :

    bash

    ssh username@192.168.1.10

5. Outils de Communication et Vérification de l'Adresse IP

Pour connaître l'adresse IP actuelle de la machine SunOS :

    Commande ifconfig :

    bash

    ifconfig -a

Cette commande montre les interfaces réseau et leurs adresses IP.
Récapitulatif

    Préparer les images ISO et le logiciel de virtualisation.
    Créer et configurer deux machines virtuelles.
    Installer SunOS et configurer les interfaces réseau avec des adresses IP différentes.
    Vérifier la connectivité réseau entre les machines (ping, SSH).

Cela vous permettra d'avoir un environnement de développement basé sur SunOS, où vous pouvez expérimenter différents services et configurations réseau.
